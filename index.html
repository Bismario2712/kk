Información sobre Desarrollo Web

Tema 1: Elementos de Páginas Web

I. Orígenes de la Web

1. Web 2.0

La Web 2.0 se refiere a la segunda generación de la web que se centra en la participación activa de los usuarios. Introdujo conceptos como redes sociales, blogs, y la colaboración en línea, permitiendo a los usuarios no solo consumir contenido sino también contribuir y compartir.

2. Web 3.0

La Web 3.0, también conocida como la Web Semántica, busca mejorar la comprensión de la información por parte de las máquinas. Utiliza tecnologías que permiten a los sistemas interpretar el contenido de manera más avanzada, facilitando la integración y el intercambio de datos entre diferentes aplicaciones.

3. La W3C y los estándares

La World Wide Web Consortium (W3C) es una organización que desarrolla estándares y directrices para asegurar el crecimiento a largo plazo de la web. Los estándares de W3C incluyen normas para HTML, CSS, y otros lenguajes web que promueven la interoperabilidad y accesibilidad de la web.

II. Lenguajes de etiquetas

1. SGML

El Standard Generalized Markup Language (SGML) es un estándar para definir lenguajes de marcado. HTML y XML son derivaciones de SGML. SGML se usa para estructurar y etiquetar documentos en un formato que puede ser leído por máquinas y personas.

2. XML

Extensible Markup Language (XML) es un lenguaje de marcado que define un conjunto de reglas para la codificación de documentos en un formato legible por humanos y máquinas. XML permite la creación de documentos con estructuras personalizadas y es ampliamente utilizado para el intercambio de datos.

3. HTML

Hypertext Markup Language (HTML) es el estándar principal para la creación de páginas web. Define la estructura del contenido web mediante el uso de etiquetas que indican cómo se debe presentar el contenido en los navegadores.

4. XHTML

Extensible Hypertext Markup Language (XHTML) es una versión más estricta de HTML que sigue las reglas de XML. XHTML busca mejorar la interoperabilidad entre navegadores al garantizar que el contenido esté bien formado y sea válido XML.

5. HTML5

HTML5 es la última versión del lenguaje HTML, que incluye nuevas características como soporte para multimedia (audio y video), gráficos (Canvas y SVG), y nuevas APIs para mejorar la funcionalidad y la experiencia del usuario en la web.

III. Otros lenguajes relacionados al desarrollo web

1. CSS/CSS3

Cascading Style Sheets (CSS) es un lenguaje utilizado para describir la presentación de documentos HTML. CSS3 es la última versión de CSS e incluye características avanzadas como animaciones, transiciones, y diseño responsivo.

2. JavaScript

JavaScript es un lenguaje de programación que se utiliza para crear contenido interactivo y dinámico en las páginas web. Permite manipular el DOM, manejar eventos, y realizar validaciones en el cliente.

3. Ajax

Asynchronous JavaScript and XML (Ajax) es una técnica para crear aplicaciones web interactivas. Permite actualizar partes de una página web de forma asíncrona sin necesidad de recargar la página completa.

4. Tecnologías y lenguajes del lado del servidor

Las tecnologías del lado del servidor, como PHP, Ruby on Rails, y Node.js, se utilizan para manejar la lógica de aplicación y la comunicación con bases de datos. A diferencia del código del lado del cliente, el código del lado del servidor se ejecuta en el servidor web.

5. Navegadores

Los navegadores web, como Google Chrome, Firefox, y Safari, son aplicaciones que interpretan el código HTML, CSS, y JavaScript para mostrar contenido web a los usuarios. Cada navegador puede tener diferencias en la forma en que renderiza las páginas web.

6. Motores renderizados

Los motores de renderizado, como Blink (Chrome) y Gecko (Firefox), son componentes de los navegadores que interpretan el código HTML y CSS para generar la representación visual de una página web en la pantalla.

IV. Herramientas de desarrollo

1. Software WYSIWYG

What You See Is What You Get (WYSIWYG) es un tipo de software que permite a los usuarios diseñar páginas web mediante una interfaz gráfica sin necesidad de escribir código. Ejemplos incluyen Adobe Dreamweaver y Microsoft Expression Web.

2. Otras alternativas

i. Figma

Figma es una herramienta de diseño colaborativo basada en la web que permite a los diseñadores crear interfaces de usuario y prototipos interactivos. Ofrece funcionalidades de diseño, prototipado, y colaboración en tiempo real.

ii. InVision

InVision es una plataforma de diseño y prototipado que permite a los diseñadores crear prototipos interactivos y recibir feedback de los usuarios. Facilita la colaboración y la revisión de diseños en equipo.

iii. Zeplin

Zeplin es una herramienta de colaboración que ayuda a los diseñadores y desarrolladores a trabajar juntos. Permite generar especificaciones y activos de diseño listos para el desarrollo a partir de archivos de diseño.

iv. GitHub

GitHub es una plataforma de desarrollo colaborativo que utiliza Git para el control de versiones. Permite a los desarrolladores almacenar y gestionar código, colaborar en proyectos, y realizar seguimientos de cambios.

3. Inspección de código desde el navegador

La inspección de código desde el navegador permite a los desarrolladores examinar y depurar el HTML, CSS, y JavaScript de una página web directamente en el navegador. Herramientas como las DevTools de Chrome o Firefox son ampliamente utilizadas para este propósito.

Tema 2: El Diseño De Una Página Web

I. Tipos de páginas Web

Los tipos de páginas web incluyen:

  • Estáticas: Páginas que muestran el mismo contenido a todos los usuarios y no cambian con frecuencia.
  • Dinámicas: Páginas que pueden cambiar el contenido en función de la interacción del usuario o datos almacenados en un servidor.
  • Interactivas: Páginas que permiten la interacción del usuario, como formularios, encuestas y aplicaciones web.
  • Responsive: Páginas que se ajustan automáticamente a diferentes tamaños de pantalla y dispositivos.

II. Puntos básicos en la elaboración de una página Web

Al elaborar una página web, es importante considerar:

  • Diseño y usabilidad: Asegurarse de que la página sea atractiva y fácil de navegar.
  • Accesibilidad: Hacer que el contenido sea accesible para personas con discapacidades.
  • Optimización para motores de búsqueda (SEO): Implementar prácticas para mejorar el ranking en los motores de búsqueda.
  • Velocidad de carga: Optimizar el rendimiento de la página para asegurar tiempos de carga rápidos.

Tema 3: Estructura De Una Página Web

I. Jerarquía de directorio de un proyecto web

La jerarquía de directorios para un proyecto web suele incluir:

  • /root: Directorio raíz del proyecto.
  • /css: Carpeta para archivos CSS.
  • /js: Carpeta para archivos JavaScript.
  • /images: Carpeta para imágenes.
  • /assets: Carpeta para otros recursos, como fuentes y videos.
  • /pages: Carpeta para páginas HTML adicionales.

II. Elementos de estructura para el diseño de una página Web: Concepto

1. Regla utilizada para validar y certificar una página web

Para validar y certificar una página web, se utiliza el W3C Markup Validation Service, que verifica el cumplimiento de las normas y estándares de HTML, CSS, y otros lenguajes web.

2. Reglas para escribir HTML

  • Las etiquetas deben estar correctamente anidadas y cerradas.
  • El atributo `alt` debe ser utilizado en las imágenes para accesibilidad.
  • Las etiquetas deben ser escritas en minúsculas.
  • El HTML debe ser bien formado y cumplir con los estándares del W3C.

3. Estructura básica de un archivo web

La estructura básica de un archivo HTML incluye:

            <!DOCTYPE html>
            <html lang="es">
            <head>
                <meta charset="UTF-8">
                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                <title>Título de la Página</title>
            </head>
            <body>
                <h1>Encabezado Principal</h1>
                <p>Contenido de la página</p>
            </body>
            </html>

5. Atributos

Los atributos en HTML proporcionan información adicional sobre los elementos. Ejemplos incluyen:

  • href (enlaces)
  • src (fuentes de medios)
  • alt (texto alternativo para imágenes)
  • class (clases para aplicar estilos)
  • id (identificadores únicos)

6. Elemento HTML

Un elemento HTML consiste en una etiqueta de apertura, contenido y una etiqueta de cierre. Por ejemplo, en <p>Este es un párrafo</p>, <p> es la etiqueta de apertura, </p> es la etiqueta de cierre, y "Este es un párrafo" es el contenido.

7. Elementos en línea y en bloque

Los elementos en línea no rompen el flujo del contenido y se muestran en la misma línea que el contenido circundante, como <span> y <a>. Los elementos en bloque, como <div> y <h1>, comienzan en una nueva línea y ocupan todo el ancho disponible.
